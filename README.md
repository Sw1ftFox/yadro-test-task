Описание задачи:
Вам нужно разработать простое Angular-приложение, которое взаимодействует с внешним API для отображения и управления данными. Приложение должно состоять из нескольких компонентов и сервисов, а также использовать маршрутизацию для навигации между страницами.

Требования:

Создание проекта:
• Используйте Angular CLI не ниже 20 версии для создания нового проекта.
• Настройте проект для использования SCSS для стилей.

Компоненты:
• Создайте компонент, который будет отображать список пользователей.
• Создайте компонент, который будет отображать подробную информацию о выбранном пользователе.
• Создайте компонент, который будет использоваться для создания и редактирования пользователей.

Сервисы:
• Создайте сервис, который будет взаимодействовать с внешним API для получения, создания, обновления и удаления пользователей.
• Используйте Angular HTTP Client для выполнения HTTP-запросов.

Маршрутизация:
• Настройте маршрутизацию для навигации между списком пользователей, деталями пользователя и формой пользователя.
• Используйте параметры маршрута для передачи идентификатора пользователя между компонентами.

Формы:
• Используйте Angular Reactive Forms для создания и редактирования пользователей.
• Реализуйте валидацию формы, включая обязательные поля и проверку формата email.

Дополнительные задачи:
• Используйте UI Kit NG-ZORRO (https://ng.ant.design/docs/introduce/en) для стилизации приложения.
• Реализуйте пагинацию для списка пользователей.
• Добавьте возможность фильтрации пользователей по имени или email.
• Деплой приложения на GitHub Pages

API:
Для выполнения задания вы можете использовать следующий API:
• Получение списка пользователей: GET https://jsonplaceholder.typicode.com/users
• Получение деталей пользователя: GET https://jsonplaceholder.typicode.com/users/{id}
• Создание пользователя: POST https://jsonplaceholder.typicode.com/users
• Обновление пользователя: PUT https://jsonplaceholder.typicode.com/users/{id}
• Удаление пользователя: DELETE https://jsonplaceholder.typicode.com/users/{id}

Критерии оценки:
• Качество кода: Чистота, читаемость и структура кода.
• Использование Angular: Правильное использование компонентов, сервисов, маршрутизации и форм.
• Взаимодействие с API: Корректное выполнение HTTP-запросов и обработка ответов.
• UI/UX: Интуитивно понятный и привлекательный интерфейс.
• Дополнительные задачи: Использование Ng-zorro, реализация пагинации и фильтрации, деплой приложения
